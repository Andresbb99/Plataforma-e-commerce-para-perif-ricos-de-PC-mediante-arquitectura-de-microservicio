## Cómo Usar este Archivo

### Tipos de Cambios

- **Added** para nuevas funcionalidades
- **Changed** para cambios en funcionalidad existente
- **Deprecated** para funcionalidad que será removida pronto
- **Removed** para funcionalidad removida
- **Fixed** para correcciones de bugs
- **Security** para vulnerabilidades

### Versioning

Este proyecto usa [Semantic Versioning](https://semver.org/):

- **MAJOR** versión cuando haces cambios incompatibles
- **MINOR** versión cuando añades funcionalidad compatible
- **PATCH** versión cuando haces correcciones de bugs

Formato: `MAJOR.MINOR.PATCH` (e.g., `1.2.3`)

### Ejemplo de Entrada

```markdown
## [1.2.0] - 2026-06-15

### Added
- Nueva funcionalidad de búsqueda avanzada
- Soporte para múltiples idiomas
- API v2 endpoints

### Changed
- Mejorado rendimiento de reportes
- Actualizado interfaz de usuario

### Fixed
- Corregido bug en autenticación
- Solucionado problema de caché

### Security
- Actualizado dependencias vulnerables
- Implementado rate limiting
```

### Etiquetas de Versión

Las etiquetas en Git siguen el formato: `v1.2.3`

```bash
git tag -a v1.2.0 -m "Release version 1.2.0"
git push origin v1.2.0
```
